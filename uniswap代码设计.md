### 4. Gas的处理
1. 对于频繁调用的函数，最好进行Gas优化， 像V3中的LowGasSafeMath库，赋值判断合并，使用NamedReturn方式。
2. EVM内部存储使用了32字节为基本单位，对于频繁的多个小单位读取及存储尽量将数据打包到一个Slot里面，就是32字节。 就像V3中Slot0一样。
3. 对于变量是Storage存储的数据，如果需要频繁调用，一定要先copy出来放在内存中进行读取，极大节省Gas费。 然后需要赋值的时候，最终在结束的时候进行赋值。
4. 对于第三点，并不是要把所有的Storage变量在使用的时候都进行内存拷贝，需要自己衡量Gas费是否合理。 比如一个Struct类型的变量，有10*32字节的大小，也是storage存储，只用了其中一个字段进行处理，完全没必要进行拷贝，只需要单独读取这个字段就好。 一次性拷贝会触发10次的SLoad，而且只用了一次，没必要。
5. 对于数据结构的涉及及变量使用的数据类型，尽量在满足业务的情况下，进行压缩数据结构，使其尽量满足32字节的倍数，充分利用EVM的存储空间。


### 5.V3 在计算tick到sqrt(price)*X96
方法： getSqrtRatioAtTick（）

price = 1.0001^tick

sqrtPrice = 1.0001^(tick/2)

sqrtPriceX96 = 1.0001^(tick/2) * 2^96

利用二进制拆分的设计：

tick = 199 = 128 + 64 + 4 + 2 + 1 = 2^7 + 2^6 + 2^2 + 2^1 + 2^0

1.0001^199 = 1.0001^128 * 1.0001^64 * 1.0001^4 * 1.0001^2 * 1.0001^1

1.0001^1   = 已知常数

1.0001^2   = 已知常数

1.0001^4   = 已知常数

1.0001^8   = 已知常数

...

1.0001^131072 = 已知常数

所以计算任意 tick 对应的价格，最多只需要 19 次乘法（因为 tick 最大 ~887272 < 2^20）。
