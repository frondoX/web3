# V2和V3的价格预言机对比

## V2的TWAP价格预言机

### 核心实现：
V2使用在Core合约中累计价格的方式，线性累计价格。 然后只记录了当前累计价格和时间戳。 price0last, price1last, blockTimestampLast. 然后无限进行下去。

在外部合约中记录了上次计算的价格时候用到的Price0， Price1， 及timestamp快照。 将最新的减去旧的就可以得到价格。 

### 问题：
1. 只有当前价格，无法获知过去的任意点的价格。
2. 必须定期调用价格累计， 因为这个是在mint/burn/swap的时候会自动调用。但当长时间没有操作时，需要激励外部Keeper进行调用，防止价格偏差过多。
3. 冷启动问题，刚部署时，链上没有上次快照信息，必须等一次更新的后，才能提供价格服务。


## V3的TWAP价格预言机

### 核心实现：
V3将价格表示为1.001的指数， Tick代表这个指数。price = 1.0001^tick。 

链上原生存储最多 65535 个历史观测点。 
```
struct Observation {
    uint32  blockTimestamp;      // 观测时间戳
    int56   tickCumulative;      // tick 累积值
    uint160 secondsPerLiquidityCumulativeX128; // 流动性加权累积
    bool    initialized;
}
Observation[65535] public observations;
```

每一个观测点只记录历史数据， 最新的观测点记录最新的数据。 一次更新只会修改最新的数据。 当永远观测点，再循环写入。

这样V3就可以查询历史价格了。

### 问题：
1. 历史价格也是少量的历史价格。 如果频繁的话， 按12s出块， 12*65535/60/60/24 = 9.1, 最多保留9天的历史价格。
2. 当长时间没有更新， 一样需要激励，保证价格的及时性。
3. 高价值项目，需要链下预言机同时使用。 chainlink。

## V3，V2
因为V2是算术平均值， 所以两个价格没有任何关联性。 必须要记两个值。
V3是几何平均， 一个价格一定是另一个价格的倒数。所以计一个即可。
